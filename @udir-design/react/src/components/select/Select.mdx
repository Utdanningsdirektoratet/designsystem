# Select

`Select` er en nedtrekksliste som lar brukeren velge ett alternativ fra et utvalg av muligheter.
Komponenten er bygget på [HTML-elementet `<select>`](https://developer.mozilla.org/en-US/docs/Web/HTML/Reference/Elements/select), som benytter seg av operativsystemets standarder.
Dette sikrer forutsigbarhet og tilgjengelighet, men det setter også noen begrensninger på hvordan komponenten kan tilpasses.
For eksempel er det ikke mulig å overstyre formateringen av alternativene.

**Passer til å**

- velge ett alternativ

**Passer ikke til å**

- navigere mellom sider eller seksjoner i en applikasjon
- velge flere alternativer, bruk heller [`Suggestion`](/docs/components-suggestion--docs) eller [`Checkbox`](/docs/components-checkbox--docs)
- velge fra en liste med få alternativer, bruk heller [`Radio`](/docs/components-radio--docs)

```tsx
import { Select, Field } from '@udir-design/react';

<Field>
  <Label>Ta et valg</Label>
  <Select>

    <Select.Option value="2">Alternativ 2</Select.Option>
    <Select.Option value="3">Alternativ 3</Select.Option>
  </Select>
</Field>;
```

### Kontrollert tilstand

Tilstanden til `Select` kan kontrolleres med `value` og `onChange`. Dette er nyttig når `Select` skal oppdateres basert på endringer i andre komponenter eller tilstander.
Du kan for eksempel filtrere alternativene i `Select` basert på brukerens valg i en annen komponent. I eksempelet under velger brukeren fylke først, slik at listen med byer blir kortere og enklere å navigere.

## Retningslinjer

### Antall alternativer

Bruk `Select` når det er 7 eller flere alternativer. Ved færre alternativer kan [`Radio`](/docs/components-radio--docs) være enklere for brukerne. Det kan også være bedre å stille ett eller flere spørsmål i forkant for å redusere antall valg.

### Tilpass bredden på feltet

Tilpass bredden etter hva brukerne skal velge, for eksempel smal bredde for telefonnummer og bredere for stedsnavn. Ulik bredde på feltene gjør det lettere å navigere i skjemaer med mange felter.

### Ikke bruk deaktivert Select

Vi unngår deaktivert (`disabled`) `Select` fordi det kan være vanskelig for brukeren å forstå hvorfor feltet ikke kan brukes. Hvis feltet ikke er relevant, bør det heller fjernes enn deaktiveres. Gi gjerne en kort forklaring på hvorfor feltet ikke er tilgjengelig.

Nav har en god forklaring på [hvorfor deaktiverte tilstander er problematisk](https://aksel.nav.no/god-praksis/artikler/deaktiverte-tilstander) og hvilke alternativer som finnes.

<DisabledEx />

### Unngå skrivebeskyttet Select

`Select` støtter `aria-readonly` for å gjøre feltet skrivebeskyttet og vise at det skiller seg fra redigerbare felt. Vi unngår likevel `aria-readonly` så langt det lar seg gjøre, fordi slike felt kan være forvirrende for noen brukere. Ikke alle forstår hvorfor innholdet ikke kan endres.

<ReadOnlyEx />

## Tekst i Select

Det skal alltid være en `label` på `Select`. I spesielle tilfeller kan labelen skjules, for eksempel i tabeller der feltet får navn fra kolonneoverskriften. Selv om labelen skjules visuelt, må den alltid være tydelig og meningsfull, siden den leses opp av skjermlesere.

- Hold `label` kort, unngå at den går over to linjer.
- Unngå å legge til beskrivelse som ikke er relevant eller meningsfull for brukeren.
- Unngå å bruke kolon etter `label`.

## Tilgjengelighet

Det er en kjent feil at HTML-elementet `<select>` ikke fungerer med VoiceOver i Chrome på macOS.

### Tastaturnavigasjon

- <kbd>Space</kbd> - åpner listen (velger også et alternativ på Mac)
- <kbd>Enter</kbd> - velger alternativ i listen (åpner også listen på Windows)
- <kbd data-icon>↓</kbd> - markerer neste alternativ i listen
- <kbd data-icon>↑</kbd> - markerer forrige alternativ i listen
- <kbd>Home</kbd> - markerer det første alternativet i listen
- <kbd>End</kbd> - markerer det siste alternativet i listen
- Andre bokstaver/tegn hopper til det første alternativet som starter på den bokstaven eller det tegnet.
