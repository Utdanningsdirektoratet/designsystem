# Pagination

`Pagination` er en liste med knapper for å navigere mellom ulike sider med innhold, for eksempel søkeresultater eller tabeller.

**Passer til å**

- navigere mellom sider med relatert innhold
- dele store datamengder opp i flere sider, for eksempel i tabeller

**Passer ikke til å**

- dele et skjema opp i flere steg – bruk heller [`FormNavigation`](/docs/components-formnavigation--docs)
- vise progresjon i en prosess – bruk heller [`ProgressBar`](/docs/components-progressbar--docs)

Komponentens aktive side styres programmatisk ved hjelp av `usePagination`.
[Les mer om `usePagination`](/docs/utilities-usepagination--docs).

```tsx
import { Pagination } from '@udir-design/react';

const onChange = (event, page: number) => console.log(`Going to page ${page}`);
const [currentPage, setCurrentPage] = useState(4);
const { pages, prevButtonProps, nextButtonProps, hasNext, hasPrevious } =
  usePagination({
    currentPage,
    setCurrentPage,
    onChange,
    totalPages: 10,
    showPages: 7,
  });

<Pagination>
  <Pagination.List>
    <Pagination.Item>
      <Pagination.Button aria-label="Forrige side" {...prevButtonProps}>
        Forrige
      </Pagination.Button>
    </Pagination.Item>
    {pages.map(({ page, itemKey, buttonProps }) => (
      <Pagination.Item key={itemKey}>
        {typeof page === 'number' && (
          <Pagination.Button {...buttonProps} aria-label={`Side ${page}`}>
            {page}
          </Pagination.Button>
        )}
      </Pagination.Item>
    ))}
    <Pagination.Item>
      <Pagination.Button aria-label="Neste side" {...nextButtonProps}>
        Neste
      </Pagination.Button>
    </Pagination.Item>
  </Pagination.List>
</Pagination>;
```

## Eksempler

### Som lenker

Knappene er som standard `<button>`-elementer. Hvis `Pagination` skal navigere til en ny URL, må knappene rendres som lenker `<a>`. Alle delkomponenter støtter `asChild`, som kan brukes til å rendre `Pagination.Button` som lenke `<a>`.

### Responsiv visning

På små skjermer er det lurt å begrense antall synlige sidetall for å spare plass. Vis kun:

- den aktive siden
- én side før og én side etter

## Retningslinjer

Bruk `Pagination` når innholdet er for omfattende for én side, for eksempel ved søkeresultater eller lange lister, og brukeren trenger å navigere frem og tilbake. Del bare opp innhold i flere sider når det forbedrer lastetid eller brukervennlighet.

Vi bruker et fast tekstmønster for `Pagination`: «Forrige» og «Neste».

Unngå derfor «Forrige side», «Neste side», «Gå tilbake», «Gå videre», «Videre» og «Tilbake».
