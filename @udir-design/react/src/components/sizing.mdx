# Sizing og spacing

Vi bruker tokens for å definere spacing (mellomrom) og sizing (bredder og høyder) på en konsistent måte. Vi benytter skalaverdier (f.eks. `size5`, `size6`, `size10`) som er relative, men som i praksis oversettes til faste verdier i `rem`. Dette gjør systemet fleksibelt og responsivt.

> 1rem tilsvarer 16px i vår standardkonfigurasjon (`data-size="medium"`).

Spacing-tokenene bygger på en skala som øker jevnt, og kan brukes til alt fra innholdsmarginer til komponent-avstand og layout-bredder.

## Generelle retningslinjer for spacing

| Bruksområde                                                                    | Token    | Verdi i px (medium) | Kommentar                                           |
| ------------------------------------------------------------------------------ | -------- | ------------------- | --------------------------------------------------- |
| Topp- og bunnmargin i toppfelt og bunnfelt                                     | `size5`  | 20px                | Gjelder også høyre/venstre-marg                     |
| Mellomrom mellom mindre innholdsseksjoner (f.eks. avsnitt, bilder og tabeller) | `size6`  | 24px                | Ekstra spacing over titler legges separat           |
| Mellomrom mellom hovedseksjoner i innhold                                      | `size10` | 40px                | Brukes f.eks. mellom seksjoner på lengre tekstsider |
| Maks bredde for tekstinnhold                                                   | –        | 800px               | For god lesbarhet på større skjermer                |
| Side-marg ved interntjenester                                                  | `size5`  | 20px                | Gjelder høyre og venstre marg                       |
| Side-marg ved publikumstjenester                                               | `size16` | 64px                | Gir mer luft og bedre tilgjengelighet               |
