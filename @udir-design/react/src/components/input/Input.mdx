# Input

`Input` er et grunnleggende skjemaelement for innsamling av brukerdata. Komponenten passer godt når du trenger full kontroll over oppsett og validering, for eksempel ved bygging av spesialtilpassede elementer.

Se også [`Textfield`](/docs/components-textfield--docs), som er en samling komponenter som ofte brukes sammen. Den kombinerer `Label`, `Description`, `Input` og `ValidationMessage`, slik at du slipper å sette dem sammen manuelt hver gang.

**Passer til å**

- skreddersy inputfelt til spesifikke behov

**Passer ikke til å**

- skrive inn lengre tekster, bruk heller [`Textarea`](/docs/components-textarea--docs)

## Slik bruker du Input

<Canvas of={InputStories.Preview} withToolbar />
<Controls of={InputStories.Preview} />

```tsx
import { Input } from '@udir-design/react';

<Field>
  <Label>Input</Label>
  <Input name="input" type="text" />
  <ValidationMessage hidden>Feilmelding</ValidationMessage>
</Field>;
```

## Eksempler

### Sett bredde med HTML size

`size`-attributtet brukes for å angi bredden på tekstfeltet. Verdien kan være et tall fra 1 til 99.

<Canvas of={InputStories.HtmlSize} />

### Bruk av forskjellige inputtyper

Trenger du full kontroll over for eksempel [`Checkbox`](/docs/components-checkbox--docs), [`Radio`](/docs/components-radio--docs) eller [`Switch`](/docs/components-switch--docs), kan du bruke `Input` med `type`-attributtet for å lage spesialtilpassede komponenter.

<Canvas of={InputStories.InputTypes} />

### Kontrollert tilstand

Tilstanden til `Input` kan kontrolleres med `value` og `onChange`. Dette er nyttig når du vil at `Input` skal oppdatere seg basert på endringer i andre komponenter eller tilstander.

<Canvas of={InputStories.Controlled} />

## Retningslinjer og tilgjengelighet

Ledeteksten og en eventuell beskrivelse skal alltid stå over `Input`. Dette gir bedre lesbarhet på små skjermer og bidrar til at ledetekst og beskrivelse ikke hindres av eventuelle feilmeldinger.

For ytterligere informasjon om tilgjengelighet og retningslinjer, se dokumentasjonen som gjelder den aktuelle inputtypen:

- [`Textfield`](/docs/components-textfield--docs)
- [`Checkbox`](/docs/components-checkbox--docs)
- [`Radio`](/docs/components-radio--docs)
- [`Switch`](/docs/components-switch--docs)
