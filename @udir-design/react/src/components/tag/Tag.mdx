# Tag

En `Tag` er en merkelapp som kan brukes til å kategorisere elementer eller kommunisere fremdrift, status eller prosess.
`Tags` kan gi brukeren et raskere overblikk over innholdet. Bruk `Tags` når innholdet har mange kategorier og det er hensiktsmessig å kunne skille mellom dem.

**Passer til å**

- kommunisere kategori, emne, status eller lignende

**Passer ikke til å**

- linke til andre sider (se [`Link`](/docs/components-link--docs))
- utføre handlinger ved klikk (bruk heller [`Button`](/docs/components-button--docs))
- la brukeren filtrere data (se [`Chip`](/docs/components-chip--docs))

## Retningslinjer

Tags bør som regel plasseres bak tittelen eller navnet på tilhørende element.

De bør ikke plasseres sammen med knapper da det kan forvirre brukeren.

## Tekst i Tag

Teksten i en `Tag` skal alltid ha stor forbokstav og små bokstaver på resten. Bruk korte etiketter, helst bare ett ord.

Bruk ord og begreper som er gjenkjennelige for løsningen, unngå ukjente og fremmede ord.

## Tilgjengelighet

`Tags` har i seg selv ingen semantisk verdi. Hvis du vil bruke flere sammen bør du derfor bruke en `<ul>` med en `<li>` for hver tag for å sikre at tagsene blir lest opp hver for seg som en liste og ikke som en setning.
